## 3. Ritual Runner Rewrite (RustClaw)

**File:** `src/ritual_runner.rs`

### 3a. Advance Model (replaced loop model)
**Before:** `run_loop()` — blocking loop with channels, timeouts, auto-skip
**After:** `advance(ritual_id, event)` — stateless, one step per call

```
advance(ritual_id, event)
  1. Load state from disk
  2. transition(state, event) → (new_state, actions)
  3. Save to disk immediately
  4. Fire-and-forget actions (notify, cleanup)
  5. If terminal/paused: return
  6. If active: spawn action in background → advance() again on completion
```

No loop, no channels, no timeout. State on disk. Process restart safe.

### 3b. State save timing
Added `save_state()` immediately after transition, before skill execution. Crash during skill no longer leaves disk state at wrong phase.

### 3c. target_root on RitualState
Added `target_root: Option<String>` to ritual state (in gid-core state_machine.rs). Set at ritual start from "Project location:" in task text. All operations (detect, skill, verify, shell) use `target_root_for(state)` — no more `detect_effective_root()` guessing.

### 3d. Verify improvements
- `run_shell` re-reads `verify_command` from `.gid/config.yml` at execution time
- `parse_verify_steps()` splits `&&` chains into labeled steps (compile, unit test, integration test)
- Error output includes which step failed: `"FAILED at step: unit test"`

### 3e. Approval routing
- `/ritual apply` aliased to `/ritual approve`
- Free-text "apply all", "approve all", "yes", "ok", "好" recognized when ritual is WaitingApproval
- Multiple WaitingApproval rituals: lists them, asks user which one
- Catch-all guard: short/ambiguous text routed to waiting ritual instead of starting new one
- `build_ritual_event_from_text()` only intercepts explicit approval patterns, not unrelated messages

### 3f. Auto-approve removed
Was: 5 min timeout → auto-skip review (lost findings)
Now: 30 min timeout → save state and exit loop. Ritual stays WaitingApproval on disk. User responds whenever.

## 5. Sub-Agent V2 (RustClaw)

**Files:** `src/agent.rs`, `src/tools.rs`, `src/ritual_runner.rs`

### New: Agent Types
4 built-in types with constant system prompts (prompt cache sharing):

| Type | Model | Tools | Max Turns |
|------|-------|-------|-----------|
| EXPLORER | sonnet | read, list, search, exec | 20 |
| CODER | opus | read, write, edit, list, search, exec | 40 |
| REVIEWER | sonnet | read, write, edit, list, search | 20 |
| PLANNER | sonnet | read, list, search | 15 |

### New: Unified `run_subagent(agent_type, task, options)`
Single entry point for all sub-agent execution. Replaces 3 separate spawn paths:
1. SpawnTool (tools.rs)
2. run_skill_as_subagent (ritual_runner.rs)
3. ApplyReview (ritual_runner.rs)

### New: TranscriptWriter
Append-only JSONL audit log per sub-agent run. Debug/audit only, not for resume.
`~/.rustclaw/transcripts/{agent_id}.jsonl`

### New: ToolRegistry::for_agent_type
Registers only the tools named in the agent type's tool list. Replaces one-size-fits-all `for_subagent`.

### Prompt Structure
System prompt = constant per type (no task/time/workspace). Everything dynamic goes in user message. Enables prompt cache sharing across same-type sub-agents.

### Design Doc
`.gid/features/subagent-v2/DESIGN.md` — full design with risks, migration plan, what changes/doesn't change.
