### 那条架构债务（必须记得）
v03-migration design §5.2 写的是 "single SQLite transaction wraps apply + checkpoint advance" 单事务。但 `GraphStore::apply_graph_delta` 内部自己开 + commit 自己的事务，没法包在外层 tx 里。T9 用了**两步 commit + 幂等重放**绕过：
1. `apply_graph_delta` 写图层（自己 commit），写入 `graph_applied_deltas(memory_id, delta_hash)` 幂等记录。
2. 单独一个小 tx 推进 checkpoint。

**Crash window**：(1) commit 后 (2) 前挂了 → 重启时 cursor 还指向上一条记录，重新 fetch 同一条 → `apply_graph_delta` 命中 `graph_applied_deltas` 行返回 `ApplyReport::already_applied_marker()` 不重写 → checkpoint 在重试中正确推进。Net: at-least-once apply, exactly-once durable effect, monotone checkpoint。设计文档里 "advance-after-commit" 这个**关键属性是保住的**——checkpoint 永远不会越过没 commit 的图写入。只是字面"single tx"被幂等行替代了。
