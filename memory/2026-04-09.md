
## recall_recent Session Startup Integration (09:10)

### Changes
- **config.rs**: Added `recent_memory_limit` field to `MemoryConfig` (default: 50, serde-defaulted)
- **memory.rs**: Added `recall_recent(limit)` method to `MemoryManager` — wraps engramai's `recall_recent`, returns `Vec<RecalledMemory>`
- **memory.rs**: Added `format_recent_for_prompt()` — formats recent memories with type tags for system prompt injection
- **agent.rs**: In `run_agent_loop()`, after `get_or_create` session, if session is **empty** (first message), calls `recall_recent(50)` and injects into system prompt
- **rustclaw.yaml**: Added `recent_memory_limit: 50` to memory config

### How It Works
1. Session starts → `session.messages.is_empty()` → true
2. Load 50 most recent memories (by created_at DESC) from engram
3. Format as `## 🧠 Recent Memories (session startup)` section
4. Append to system prompt **after** per-message auto-recall context
5. On subsequent messages in same session, `messages` is non-empty → skip

### Key Design Decisions
- **Session-emptiness check, not hook**: One-time startup action, not per-message
- **After auto-recall, not instead of**: Both injected. Auto-recall = query-relevant; recent = temporal continuity
- **50 default**: ~5-10k tokens, configurable via `recent_memory_limit`
- **No truncation**: Each stored memory already Haiku-distilled

### Public API

- **`relation_weight(relation) -> f64`** — edge relation → weight mapping
- **`ClusterConfig`** — teleportation_rate, num_trials, min_community_size, hierarchical, seed
- **`build_network(graph) -> (Network, Vec<String>)`** — file-level network from code graph
- **`run_clustering(net, idx_to_id, config) -> (Vec<RawCluster>, ClusterMetrics)`** — flat/hierarchical Infomap
- **`map_to_components(clusters, graph) -> ClusterResult`** — raw clusters → Node/Edge
- **`auto_name(file_paths) -> String`** — common-prefix directory naming
- **`cluster(graph, config) -> Result<ClusterResult>`** — main entry point

### Key Decisions
- Orphan reassignment: strongest out-neighbor by edge weight, fallback to singleton "misc"
- Hierarchical: dot-notation IDs (`infer:component:0.1.3`), parent members include all descendants
- auto_name avoids generic dirs like "src", uses most frequent dir when no common prefix
- `map_to_components` metrics are placeholder (overwritten by caller from `run_clustering`)
