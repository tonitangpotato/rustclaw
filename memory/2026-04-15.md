potato 想把 gid 发展为通用知识图谱工具。深入读了 engram 和 gid 源码后发现：

- 两个 crate 的聚类能力重叠但互相不知道（engram: 4信号connected-component; gid: Infomap加权网络）
- Infomap 不绑定代码——代码特定的只是边权策略，换边权就能通用
- engram extractor 已有 LLM text→fact pipeline，改输出格式为三元组就能喂 gid graph
- 通用 KB pipeline: text → engram extractor(三元组) → gid graph → Infomap聚类 → 社区 → engram recall加权
- 战略意义：认知记忆层(engram) + 结构知识层(gid) 双层配合，市面上没人这么做
