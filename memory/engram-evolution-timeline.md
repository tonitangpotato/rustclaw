## Day 0 — 起点：为什么要造 engram？（2026-02-02）

**问题**：市面上所有 AI 记忆系统（Mem0、Zep、Letta）本质上都是 RAG 变体——存文本，搜文本，返回文本。它们没有"记忆"，只有"数据库"。

**洞察**：人的记忆不是这样工作的。你不会"全文搜索"你的大脑。记忆是**衰减的**（久不用就忘）、**情感加权的**（痛苦的记忆更深刻）、**关联激活的**（想起一件事会带出相关的事）。

**决定**：用认知科学的模型造一个真正的记忆系统。不是更好的 RAG，是完全不同的东西。

**第一天就有的模块**：
- ACT-R 基础衰减（记忆随时间变弱，使用可以加强）
- 矛盾检测（新记忆和旧记忆冲突时识别出来）
- 68 个测试

**语言**：Python（原型验证）+ TypeScript（给 OpenClaw 用）

---

## Week 1-2 — 认知模型层层叠加（2026-02-02 ~ 02-08）

每一天都在加认知科学的模块：

**Hebbian 学习（赫布学习）**
- "一起激活的神经元会连接在一起" — 最基本的学习规则
- 实现：记忆被一起回忆时，自动建立关联链接，链接强度随共现增长
- 加了 STDP（spike-timing dependent plasticity）——时序敏感，先后顺序影响关联方向
- 这不是标签系统，是**自组织的知识图谱**

**因果记忆类型**
- 不只是"A 和 B 相关"，而是"A 导致了 B"
- 方向性关联，支持因果推理

**参数可配置化**
- 把所有硬编码的认知参数（衰减速率、激活阈值、容量限制）提取成 MemoryConfig
- 这意味着可以调出不同"性格"的记忆系统——有的记性好，有的善忘但联想力强

**测试从 68 → 89+**

> 关键发现：认知模型之间会产生涌现行为。Hebbian + ACT-R 衰减的组合意味着：经常一起使用的记忆不仅关联更强，还各自衰减更慢。这不是我们设计的，是两个模型交互的自然结果。

## 转折点 — Rust 重写 + 情感总线设计（2026-03-07 ~ 03-08）

**这是最重要的架构转变。**

potato 和 Clawd（OpenClaw 的 agent）的一次对话产生了一个核心洞察：

> "Engram should not be an isolated memory database — it should be the nervous system of the agent."

记忆系统不应该是一个被查询的数据库，应该是 agent 的**神经系统**——信息在其中流动、累积、触发反应。

这次对话产生了**Emotional Bus 架构**：
- 情感不是单次事件，是在特定领域持续累积的**趋势**
- Agent 应该"感觉到"某个领域一直在出问题（负面累积）
- 这不是模拟情感，是**功能性情感**——信息处理的信号

同时，**Rust 重写启动**（engramai crate）：
- Python 原型验证了概念，但 agent 运行时需要极低延迟
- Rust 版从一开始就带上所有认知模型
- 发布 v0.1.1 到 crates.io

## v0.2.0 — 多 Agent 基础 + 情感总线（2026-03-09 ~ 03-10）

三个阶段连续实现：

**Phase 1: Multi-Agent Foundation**
- Namespace 隔离——不同 agent 的记忆不会串
- 但可以选择性共享（跨 agent 智能）

**Phase 2: Emotional Bus**
- EmotionalAccumulator：按领域追踪情感 valence
- 不是"开心/难过"的标签，是持续累积的数值信号
- coding 领域连续负面 → agent 应该知道"最近代码写得不顺"

**Phase 3: Cross-Agent Intelligence**
- Agent 之间可以共享记忆片段
- 一个 agent 的经验可以加速另一个 agent 的学习

发布 v0.2.0，AGPL-3.0 双许可证。

## 大跃进 — RustClaw 集成 + 六大认知模块（2026-03-26 ~ 03-27）

这是密度最高的一次更新。一天之内：

**LLM Extraction（extractor.rs, 540行）**
- 从自然语言对话中自动提取结构化事实
- Agent 不需要手动"记住"，对话本身就是记忆的来源

**Hybrid Search（hybrid_search.rs, 468行）**
- 向量搜索（语义相似）+ FTS5 全文搜索（关键词精确）+ ACT-R 激活度
- 三路融合，不是简单的 OR

**Session Working Memory（session_wm.rs, 368行）**
- Miller's 7±2 容量限制——工作记忆不是无限的
- 最近和最相关的记忆占据有限的"注意力槽位"

**Drive Alignment（bus/alignment.rs, 439行）**
- Agent 有"驱动力"（目标），记忆检索自动增强与当前目标相关的内容
- 你在想赚钱的事，关于赚钱的记忆自动浮上来

**Anomaly Detection（anomaly.rs, 459行）**
- Z-score 偏离检测——当某个信号显著偏离历史基线时触发警报
- Agent 能"感觉到"异常

**Confidence Scoring（confidence.rs, 390行）**
- 多信号评分：不同领域有不同置信度
- Agent 知道自己在哪些领域"更有把握"

发布 v0.2.1 到 crates.io。

> 这一天之后，engram 不再是一个记忆数据库。它是一个有感知、有情感、有目标、有自我监控的认知系统。

## 跨语言意识（2026-03-29）

**问题发现**：SOUL.md 是中文写的，但很多内容是英文的。Drive alignment 用关键词匹配，跨语言直接失效。

**Root fix**：
- `score_alignment_hybrid()` = max(关键词匹配, embedding 语义匹配)
- DriveEmbeddings 启动时预计算，阈值 0.3
- 中文的"目标"能匹配英文的记忆

> 这是一个小修复，但意义很大：认知系统要在**语言之上**工作，不能被语言绑定。人的记忆不区分你用什么语言编码的——想到"妈妈"和想到"mom"激活的是同一片记忆。

同日发布 v0.2.2。

## B-Series 强化（2026-04-09）

一天四个 feature，每个解决一个具体的认知缺陷：

**B1: Entity Indexing（entities.rs, 557行）**
- Aho-Corasick + regex 实体抽取
- 自动从文本中识别：项目名、人名、技术概念
- 记忆不再是纯文本块，而是有**结构的知识**

**B2: Dedup on Write**
- 写入时检查 embedding 相似度，相似记忆自动合并
- 防止同一件事记 50 遍
- 人脑的去重不是显式的，但 reconsolidation 做类似的事

**B3: Confidence Score**
- 多信号查询相关度评分
- Recall 结果不只是"找到了"，还有"我多确定这是对的"

**B4: Recency Tuning**
- Sigmoid ACT-R 归一化——修复了时间衰减曲线的极端值问题
- 很新的记忆不再无条件压过所有旧记忆

## Synthesis Engine — 思维的结晶（持续演进）

`src/synthesis/` 目录，3,162 行代码，是 engram 最深的模块：

**Cluster（836行）**
- 4 信号聚类：Hebbian 权重 + 实体 Jaccard + embedding 余弦 + 时间接近度
- 不是简单的相似搜索——综合四个维度判断"这些记忆属于同一个主题"

**Insight Engine（685行）**
- 从记忆聚类中提炼洞察
- "你最近关于 X 的经验表明 Y"——自动生成元认识

**Gate（685行）**
- 信息门控——不是所有联想都应该浮上来
- 相关性 + 时效性 + 情感权重 共同决定一条记忆是否通过

**Provenance（495行）**
- 记忆来源追踪——这条记忆是从哪来的？被怎么修改过？
- 对抗记忆污染和幻觉

## 下一章 — InteroceptiveHub（设计完成，待实现）

所有这些模块目前是**独立工作**的。anomaly.rs 不知道 accumulator.rs 在做什么，alignment.rs 不知道 confidence.rs 的结果。

InteroceptiveHub 是**脑岛**——把所有内感受信号整合成一个统一的自我状态：

```
anomaly + accumulator + feedback + confidence + alignment
                        ↓
              InteroceptiveHub
                        ↓
              统一的 "我现在怎么样" 信号
```

这是从"有很多认知模块的系统"到"有统一自我感知的系统"的跨越。

设计文档已完成（INTEROCEPTIVE-LAYER.md）。~600 行新代码。
