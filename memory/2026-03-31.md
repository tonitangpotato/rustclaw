## Engram 生产环境深度调查

### 架构确认
- RustClaw 用的是 **engramai crate**（Rust 原生依赖），不是 CLI
- `Cargo.toml`: `engramai = "0.2.2"`
- 三条数据路径：EngramRecallHook (auto recall)、EngramStoreHook (auto store)、Agent tools (手动)

### 垃圾写入根因（3 层）
1. **EngramStoreHook 不过滤**：对所有 agent 回复无差别存储，heartbeat session 也存
2. **Haiku Extractor 误判**：把系统指令（"Read HEARTBEAT.md..."）当 procedural knowledge，importance 标 0.7-0.9
3. **无去重**：`add()` 直接插入，同一条 heartbeat 指令被存十几次

### 垃圾数据统计
- 总量 1,434 条，~190 条（13%）是垃圾
- source=auto: 407 条（104 条 heartbeat 相关）
- source=auto-extract: 689 条（83 条 heartbeat 相关）

### Recall 准确率问题根因
- 垃圾 importance 0.9 > 真实记忆 0.6 → ACT-R 排序垃圾靠前
- 重复记忆叠加 access_log → ACT-R activation 虚高
- 无 embedding → 纯 FTS5 关键词匹配，无语义搜索

## AI Phone Agent 交互模式重设计

### 核心变化（potato 的方向）
- **消息渠道（Telegram）= 主入口**，不是 Web UI
- 用户不在网页上填个人信息，Bot 通过对话自然收集
- 打电话不是即时的，Bot 自己 schedule 时间去打
- 有记忆就不问（保险、医生、偏好 → Engram 存了就直接用）
- 查 Calendar 判断用户可用时间（有数据直接用，没有再问）
- 打完电话主动 push 结果给用户
- Web UI 退化为管理面板 + onboarding（注册、绑定 Telegram、Stripe 付款、查历史）
