### 三层缺陷(potato 的直觉 + 多轮讨论收敛)

**1. Store 层** — 关键 meta 事件(如 "canonical repo 是 engram,老的 engramai 已 consolidated")没有被作为高优先级规范记忆存入,或没被 knowledge_compile 编译成 topic。

## 2026-04-23 17:00 EDT — ISS-021 Phase 5b complete (counterfactual measurement)

**Decision framework used**: potato was tired and delegated the design decision to me. I had two interpretations of the Phase 5b spec:
- A (literal): copy prod DB → migrate it → rerun baseline fixtures against it. Fatal flaw: fixtures' `relevant_ixs` point at synthetic strings not present in prod → P@3 ≡ 0, no signal.
- B (controlled experiment on synthetic corpus): for each fixture, run polluted variant (corpus items get 80-token Telegram header prefix) and clean variant, same query, same recall path. delta = P_clean − P_polluted directly isolates header pollution as a variable.

**Picked B** because the hypothesis under test ("embedding model contaminated by channel headers") is a *model property*, not a data-distribution property. Controlled experiment on synthetic corpus measures the model property directly and cleanly. Also: B takes 1 hour, A takes 1.5 days with zero useful output.

**Result**: delta = +0.000 across 10/10 fixtures. Gate threshold was 0.15. Hypothesis falsified.

**Harness validity double-checked** (important — a delta=0 result is only interesting if the harness could detect a real effect):
- Regime diagnostic self-reports `EMBEDDING (Ollama/nomic-embed-text live)` — confirms embedding path active, not FTS fallback.
- Sanity probe `recall_harness_sanity_reacts_to_large_content_changes`: ~150-token off-topic prefix drops P@3 from 0.767 → 0.333 (drop=0.433). Harness is content-sensitive; a real effect would be caught.
- Storage audit: 20 runs × 8 items stored, 0 quarantined. Populations identical across modes.

**Interpretation**: nomic-embed-text, as a mean-pooled transformer embedding, treats a fixed prefix shared by every corpus item as common-mode — it cancels out of pairwise cosine similarities. Matches retrieval literature. `Architectural` fix (Phase 2+3 envelope side-channel) is still correct on its own merits (single source of truth, clean extractor input, structured metadata) — but not because of recall quality.
