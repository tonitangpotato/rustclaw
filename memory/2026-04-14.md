## GID 作为通用 Knowledge Graph 引擎（讨论）

**核心洞察**：AI 时代 KB 的 consumer 是 agent，不是人。KB 是领域化、专业化的 context 提供给 AI。

**gid vs engram 定位**：
- engram = agent 自己的记忆（经历、学习、有时间维度）
- gid/KB = 外部的领域知识结构（参考资料，多 agent 共享，不属于任何一个 agent）

**gid 做通用 KG 的优势**：
- 已有：图引擎 + SQLite backend + Infomap 聚类 + LLM→graph 双向管道 + 影响分析 + 可视化
- 比 Neo4j 轻，比 LightRAG/GraphRAG 图能力强
- Code graph 只是一个 vertical application

**需要泛化**：
1. API 命名/接口（node_type 不限于代码类型，relation 不限于代码关系）
2. 通用 text → entity/relation extraction pipeline（接 LLM）
3. 多跳 path query API

**状态**：讨论阶段，之后有空再推进
