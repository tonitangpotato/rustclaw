## Content

RAG is great, until it isn't.

Mintlify's assistant could only retrieve chunks of text that matched a query. If the answer lived across multiple pages, or the user needed exact syntax that didn't land in a top-K result, it was stuck. They wanted it to explore docs the way you'd explore a codebase.

### Problem
- Sandbox approach: p90 session creation ~46 seconds (GitHub clone + setup)
- 850K conversations/month → micro-VMs cost >$70K/year (1 vCPU, 2 GiB RAM, 5-min session)

### Solution: ChromaFs
A **virtual filesystem** that intercepts UNIX commands and translates them into queries against their existing Chroma vector database.

- Session creation: ~46s → **~100ms**
- Marginal per-conversation compute cost: **zero** (reuses existing Chroma infra)

### Architecture
- Built on **just-bash** (Vercel Labs) — TypeScript reimplementation of bash
- `just-bash` exposes a pluggable `IFileSystem` interface
- ChromaFs implements this interface, translating filesystem calls → Chroma queries

### Key Design Decisions

1. **Directory Tree Bootstrapping**: Entire file tree stored as gzipped JSON (`__path_tree__`) in Chroma. On init, decompresses into `Set<string>` (files) + `Map<string, string[]>` (dirs→children). `ls`, `cd`, `find` resolve in-memory, no network.

2. **Access Control**: `isPublic` and `groups` fields in path tree. ChromaFs prunes tree based on user permissions before building — per-user RBAC with no Linux user groups or container images.

3. **Page Reassembly**: Pages chunked for embedding. `cat /auth/oauth.mdx` → fetches all chunks with matching page slug, sorts by `chunk_index`, joins. Results cached.

4. **Lazy File Pointers**: Large OpenAPI specs in S3 only fetched on `cat` access.

5. **Read-Only**: All write operations throw EROFS. Stateless, no cleanup, no corruption risk.

6. **Smart grep**: Intercepts `grep -r`, translates to Chroma query (`$contains` for fixed strings), gets matching slugs, `bulkPrefetch` into Redis cache, then narrows grep to only matched files for in-memory fine filter. Large recursive queries complete in milliseconds.
